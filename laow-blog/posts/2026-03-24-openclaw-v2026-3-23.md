# OpenClaw v2026.3.23 发布：大量重要修复上线

## 版本概览

OpenClaw 于 2026年3月23日 发布了 v2026.3.23 正式版本，这是近期修复最为密集的版本之一。

**共修复了 23+ 个问题**，涵盖浏览器自动化、认证系统、多平台消息通道、模型供应商等多个模块。

---

## 核心修复亮点

### 🔧 浏览器自动化 (Browser/MCP)

| 问题 | 影响 |
|------|------|
| macOS Chrome attach 超时 | 用户配置文件超时减少 |
| Linux headless 浏览器启动回归 | 第二次运行浏览器启动问题修复 |
| 复用已有浏览器循环 | 减少重复启动，提升稳定性 |

### 🔐 ClawHub 认证修复

- **macOS auth**: 正确使用 macOS auth 配置和 XDG auth 路径
- **登录状态保持**: 技能浏览不再静默回退到未认证模式
- **修复 429 错误**: 解决 ClawControl 因认证 token 问题导致的 Rate Limit

### 💬 多平台消息修复

**Discord**:
- 修复原生斜杠命令的未授权回复
- components 字段恢复为可选

**Feishu (飞书)**:
- `message(..., media=...)` 文件/图片发送修复
- pin/unpin/react 操作不再报 schema 验证错误

**Telegram**:
- 修复同聊天室入站消喘顺序问题
- `asDocument` 作为 `forceDocument` 别名支持

**WhatsApp**:
- `openclaw doctor --fix` 不再写入无效的插件白名单条目

### 🤖 模型供应商修复

| 供应商 | 修复内容 |
|--------|----------|
| OpenAI Codex OAuth | 代理环境下 token 刷新成功 |
| Mistral | 降低 max-token 默认值，修复 422 错误 |
| DeepSeek | 重构为共享单提供商插件入口 |
| OpenRouter | 修复 auto 路由定价缓存递归问题 |
| MiniMax OAuth | 代理环境下 OAuth 登录恢复 |

### 🛡️ 安全修复

- **exec approvals**: 拒绝单引号 `$0`/`$n` token，防止恶意变量展开
- **Gateway/auth**: Canvas 路由需要认证，admin scope 才能重置会话

### 🗄️ 其他重要修复

- **memory-lancedb**: 插件运行时状态引导修复
- **Gateway/supervision**: 防止 launchd/systemd 下的崩溃循环
- **Agents/subagents**: 快速完成的 worker 不再报告为超时
- **Agents/Anthropic**: preserve thinking block ordering
