# 项目管理目录

## 目录结构

```
projects/
├── docs/             # 文档
│   ├── 01-需求/    # 需求阶段
│   ├── 02-设计/    # 技术设计阶段
│   ├── 03-测试/    # 测试阶段
│   ├── 04-开发/    # 开发阶段
│   └── 05-记录/    # 过程记录
├── code/            # 源代码
└── deploy/          # 部署包、配置
```

## 项目开发流程

### 流程概述
```
需求阶段 → 设计阶段 → 开发阶段 → 测试阶段
  ↑                            ↓
  ←←←←←← 变更记录 ←←←←←←←←←←
```

### 1. 项目启动
新项目启动前，必须先按「目录结构」搭建完整的项目管理目录。

### 2. 需求阶段
- 负责人与我（强哥）多次交流，收集并分析用户需求
- 我会将强哥视为客户，有任何不清楚的地方必须提问澄清
- 整理后形成完整的需求文档，发送给强哥确认
- **确认通过后**方可进入下一阶段

**产出文档：**
- `01-需求/需求.md`
- `01-需求/功能清单.md`
- `01-需求/验收标准.md`

### 3. 设计阶段
- 输入为需求阶段的全部文档
- 根据需求、功能清单进行技术方案、接口设计、数据结构设计
- 所有设计文档必须可溯源，能在需求文档中找到对应依据
- 设计完成后发送给强哥确认
- **确认通过后**方可进入下一阶段

**产出文档：**
- `02-设计/技术方案.md`
- `02-设计/接口设计.md`
- `02-设计/数据结构.md`

### 4. 开发阶段
- 根据需求和设计文档进行编码实现
- 提前编写测试用例，覆盖功能、接口和验收标准的所有内容
- 每个阶段完成后在 `04-开发/技术笔记.md` 中记录遇到的问题和解决方案
- 开发过程中如有 Skills 增加或修改，同步更新 `技术方案.md` 中的 Skills 清单
- 开发完成后通知强哥

**产出文档：**
- `04-开发/开发计划.md`
- `04-开发/技术笔记.md`
- Skills 代码（存入 `code/` 目录）

### 5. 测试阶段
- 编码完成后执行测试，将结果写入测试报告并发送给强哥
- 测试用例必须覆盖验收标准的全部内容

**产出文档：**
- `03-测试/测试用例.md`
- `03-测试/测试报告.md`

### 6. 变更管理
- 项目过程中如有需求、设计或开发的变更，必须记录到 `05-记录/变更记录.md`
- 变更后需同步更新 `01-需求`、`02-设计`、`04-开发` 中关联的文档
- 保证所有文档的关联性和一致性
- **文档是项目交付的一部分，最终必须与系统功能保持一致**

### 重要原则
1. 每个阶段完成后必须经强哥确认，方可进入下一阶段
2. 设计阶段必须严格遵守需求阶段的设计，不得随意偏离
3. 变更必须同步更新所有关联文档，保证文档的准确性
4. 测试用例必须覆盖验收标准的全部内容
5. 开发过程中遇到的问题和解决方案必须记录到技术笔记
6. Skills 如有新增或修改，必须同步更新技术方案中的 Skills 清单
7. **文档是项目交付的一部分，最终必须与系统功能保持一致**
