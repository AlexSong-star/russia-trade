# 接口设计

## 一阶段 vs 二阶段 接口对比

| 维度 | 阶段一 | 阶段二 |
|------|--------|--------|
| 接口风格 | Python工具脚本 | RESTful API |
| 认证 | 无 | JWT Token |
| 调用方式 | 命令行/脚本 | HTTP请求 |
| 文档 | 代码注释 | Swagger/OpenAPI |

---

## 阶段一：工具脚本接口

### 定位
- 由老六手动调用
- 输出结果写入飞书表格
- 无需API认证

### 工具清单

| 工具 | 语言 | 调用方式 | 输入 | 输出 |
|------|------|----------|------|------|
| search_tool.py | Python | 命令行 | 关键词 | Excel |
| research_tool.py | Python | 命令行 | 公司名 | Excel |
| contact_tool.py | Python | 命令行 | 公司名 | Excel |
| letter_tool.py | Python | 命令行 | 客户ID | Excel |

---

### 1.1 搜索工具 search_tool.py

```python
# 调用方式
python search_tool.py --keyword "Огнеупоры для МНЛЗ" --channel b2b_center

# 输出
customers.xlsx  # 客户基本信息Excel
```

---

### 1.2 背调工具 research_tool.py

```python
# 调用方式
python research_tool.py --company "Северсталь" --website "https://severstal.com"

# 输出
research.xlsx  # 背调信息Excel
```

---

### 1.3 联系人深挖工具 contact_tool.py

```python
# 调用方式
python contact_tool.py --company "Северсталь" --domain "severstal.com"

# 输出
contacts.xlsx  # 联系人Excel
```

---

### 1.4 开发信生成工具 letter_tool.py


## 阶段二：RESTful API 接口

### API 基础信息

| 项目 | 值 |
|------|------|
| 基础URL | `https://api.example.com/v1` |
| 认证 | JWT Bearer Token |
| 格式 | JSON |
| 编码 | UTF-8 |

---

### 2.1 认证接口

#### POST /auth/register - 用户注册
```json
// Request
{
    "username": "客户名",
    "email": "客户@邮箱.com",
    "password": "密码",
    "company_name": "XX贸易公司"
}

// Response 201
{
    "user_id": "uuid",
    "access_token": "jwt-token",
    "expires_in": 86400
}
```

#### POST /auth/login - 用户登录
```json
// Request
{
    "email": "客户@邮箱.com",
    "password": "密码"
}

// Response 200
{
    "access_token": "jwt-token",
    "expires_in": 86400
}
```

---

### 2.2 客户管理接口

#### GET /customers - 获取客户列表
```json
// Query Params
?page=1&page_size=20&customer_type=A&status=待联系

// Response 200
{
    "total": 100,
    "page": 1,
    "page_size": 20,
    "data": [
        {
            "customer_id": "uuid",
            "company_name": "Северсталь",
            "website": "https://severstal.com",
            "customer_type": "A",
            "source_channel": "b2b_center",
            "development_status": "待联系",
            "created_at": "2026-03-23T10:00:00Z"
        }
    ]
}
```

#### POST /customers - 新增客户
```json
// Request
{
    "company_name": "Северсталь",
    "website": "https://severstal.com",
    "source_channel": "b2b_center"
}

// Response 201
{
    "customer_id": "uuid",
    "message": "创建成功"
}
```

#### GET /customers/{customer_id} - 获取客户详情
```json
// Response 200
{
    "customer_id": "uuid",
    "company_name": "Северсталь",
    "website": "https://severstal.com",
    "address": "俄罗斯钢铁街1号",
    "scale": "大型",
    "employee_count": 50000,
    "has_mnlz": true,
    "procurement_mode": "集中",
    "customer_type": "A",
    "contacts": [...],
    "letters": [...]
}
```

#### PATCH /customers/{customer_id} - 更新客户
```json
// Request
{
    "development_status": "已联系",
    "notes": "客户有意向"
}

// Response 200
{
    "customer_id": "uuid",
    "message": "更新成功"
}
```

## 阶段一 vs 阶段二 接口差异总结

| 功能 | 阶段一接口 | 阶段二API |
|------|-----------|-----------|
| 客户管理 | Excel + 飞书表格 | REST API |
| 搜索触发 | 命令行脚本 | POST /tasks/search |
| 开发信生成 | 手动调用脚本 | POST /letters/generate |
| 发送 | 客户手动发送 | 自动发送 + API记录 |
| 配置管理 | 修改代码 | POST /channels, /keywords |
| 状态查看 | 打开飞书表格 | GET /customers, /send-logs |
