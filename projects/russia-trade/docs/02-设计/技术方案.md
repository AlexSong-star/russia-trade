# 技术方案

## 一阶段 vs 二阶段 技术架构对比

| 维度 | 阶段一（数据交付） | 阶段二（SaaS平台） |
|------|-------------------|-------------------|
| 实现方式 | OpenClaw Skill | Web应用 |
| 数据存储 | 飞书多维表格 | 数据库 |
| 搜索执行 | 老六手动+工具辅助 | 自动定时任务 |
| 开发信 | 老六生成，客户确认 | 自动生成+发送 |
| 交付物 | Excel/飞书表格 | Web管理后台 |
| 部署 | 无需部署 | VPS/服务器 |

## 阶段一技术方案（数据交付）

### 核心定位
- **执行者**：老六（我）
- **目标**：快速验证，交付数据给客户

### 数据充实流程（4个Skills串联）

```
Searcher（搜索）
    ↓ 爬基础数据（公司名、渠道链接）
Researcher（背调）
    ↓ 补全字段（地址、规模、主营、设备、招标等）
Contacts（联系人深挖）
    ↓ 找到采购负责人（姓名、职位、邮箱、电话）
Emailer（开发信）
    → 生成俄语开发信，发送，状态流转
```

| Skill | 文件夹 | 功能 |
|-------|--------|------|
| russia-trade-searcher | code/russia-trade-searcher | 从b2b-center/metaprom/Yandex搜索客户基础数据 |
| russia-trade-researcher | code/russia-trade-researcher | 公司背调：访问官网+Yandex，补充地址/规模/主营/设备/招标字段 |
| russia-trade-contacts | code/russia-trade-contacts | 联系人深挖：找采购负责人姓名、邮箱、电话 |
| russia-trade-emailer | code/russia-trade-emailer | 生成俄语开发信，状态流转管理 |

**数据在Skills间的流转**：飞书多维表格（客户主表）作为唯一数据源，各Skill读写同一张表
- **自动化程度**：辅助工具为主，人工判断为核心

### 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 编程语言 | Python 3.10+ | 个人电脑运行 |
| 网页抓取 | Playwright / requests | 浏览器自动化/请求 |
| 搜索引擎 | Yandex（Playwright自动化） | 俄语搜索，无需API |
| 数据存储 | 飞书多维表格 | 我来维护 |
| 字段变更 | 客户主表新增"渠道链接"字段 | 所在渠道之后，存储为文本（解决飞书超链接字段写入API限制） |
| 推送 | 飞书消息 | 每日报告推送 |

### 阶段一依赖的 OpenClaw Skills

> 以下 Skills 用于实现阶段一功能。标注**需开发**的是项目专属技能，标注**现有**的是 ClawHub/系统已有的。

| Skill 名称 | 功能 | 说明 |
|------------|------|------|

| `russia-trade-researcher` | 公司背调工具集 | 采集官网信息、Yandex Maps 地址等 |
| `russia-trade-contact-finder` | 联系人深挖工具集 | 官网表单、LinkedIn、Hunter.io 等 |
| `russia-trade-letter-generator` | 开发信生成工具 | LLM 生成俄语开发信 |

#### 现有的 Skills（系统已有）

| Skill 名称 | 来源 | 功能 |
|------------|------|------|
| `feishu-bitable` | ClawHub | 飞书多维表格操作 |

| `agent-browser` | ClawHub | 浏览器自动化（Playwright） |

#### 换电脑安装清单

```bash
# 1. 安装项目专属 Skills
# russia-trade-searcher     （需开发）
# russia-trade-researcher    （需开发）
# russia-trade-contact-finder（需开发）
# russia-trade-letter-generator（需开发）

# 2. 安装系统已有 Skills（从 ClawHub）
skill install feishu-bitable
skill install feishu-calendar
skill install agent-browser

# 3. 安装 Python 依赖
pip install playwright requests beautifulsoup lxml deep-translator pandas

# 4. 安装 Playwright 浏览器
playwright install chromium
```

### 阶段一交付物

1. **客户清单**（飞书表格）
   - 公司名称、官网、邮箱、电话
   - 背调信息
   - 客户分类（A/B/C）
   - 联系人信息

2. **开发信清单**（飞书表格）
   - 每位客户的定制开发信
   - 发送状态（待发送/已发送）

3. **每日报告**（飞书消息）
   - 新客户数量
   - 重点客户摘要

## 阶段二技术方案（SaaS平台）

### 核心定位
- **使用者**：客户（贸易商）
- **目标**：交付完整产品，客户自助使用
- **自动化程度**：全自动化

### 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 后端 | Python FastAPI | Web服务 |
| 前端 | React + Ant Design | 管理后台 |
| 数据库 | PostgreSQL | 客户数据 |
| 缓存 | Redis | 队列缓存 |
| 定时任务 | APScheduler / Celery | 自动化任务 |
| 网页抓取 | Playwright | 搜索/背调 |
| 邮件 | SendGrid | 邮件发送 |
| WhatsApp | WhatsApp Business API | WhatsApp发送 |
| Telegram | Telegram Bot API | Bot发送 |
| 部署 | Docker + Nginx | VPS部署 |
