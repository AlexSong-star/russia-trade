## 阶段二：数据库设计（PostgreSQL）

### 表规划

| 表名 | 说明 |
|------|------|
| users | 用户表（客户账号） |
| customers | 客户主表 |
| contacts | 联系人表 |
| letters | 开发信表 |
| send_logs | 发送记录表 |
| channels | 渠道配置表 |
| keywords | 关键词配置表 |
| send_queue | 发送队列表 |

---

### 2.1 用户表 users

```sql
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    company_name VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE
);
```

---

### 2.2 客户主表 customers

```sql
CREATE TABLE customers (
    customer_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(user_id),  -- 所属用户
    company_name VARCHAR(255) NOT NULL,
    company_name_en VARCHAR(255),
    website VARCHAR(500),
    customer_type VARCHAR(10) CHECK (customer_type IN ('A', 'B', 'C')),
    source_channel VARCHAR(50),
    source_url VARCHAR(500),
    address TEXT,
    scale VARCHAR(20) CHECK (scale IN ('大型', '中型', '小型')),
    employee_count INTEGER,
    main_products TEXT[],  -- 数组类型
    has_mnlz BOOLEAN DEFAULT FALSE,
    procurement_mode VARCHAR(20) CHECK (procurement_mode IN ('集中', '分厂')),
    financial_status VARCHAR(20) CHECK (financial_status IN ('上市', '非上市')),
    recent_tenders TEXT[],
    development_status VARCHAR(20) DEFAULT '待联系' 
        CHECK (development_status IN ('待联系', '已联系', '已回复', '成交', '无效')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, company_name)  -- 同一用户下公司名不重复
);
```

---

### 2.3 联系人表 contacts

```sql
CREATE TABLE contacts (
    contact_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    name VARCHAR(100),
    title VARCHAR(100),
    title_en VARCHAR(100),
    email VARCHAR(255),
    phone VARCHAR(50),
    telegram VARCHAR(100),
    whatsapp VARCHAR(50),
    linkedin_url VARCHAR(500),
    source_method VARCHAR(50),
    validity VARCHAR(20) DEFAULT '待确认' 
        CHECK (validity IN ('有效', '无效', '待确认')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### 2.4 开发信表 letters

```sql
CREATE TABLE letters (
    letter_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    contact_id UUID REFERENCES contacts(contact_id) ON DELETE SET NULL,
    content TEXT NOT NULL,
    generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    confirmed_at TIMESTAMP,
    status VARCHAR(20) DEFAULT '待确认' 
        CHECK (status IN ('待确认', '已确认', '已发送')),
    confirmed_by UUID REFERENCES users(user_id)
);
```

---

### 2.5 发送记录表 send_logs

```sql
CREATE TABLE send_logs (
    log_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    letter_id UUID REFERENCES letters(letter_id) ON DELETE SET NULL,
    customer_id UUID REFERENCES customers(customer_id) ON DELETE SET NULL,
    contact_id UUID REFERENCES contacts(contact_id) ON DELETE SET NULL,
    channel VARCHAR(20) CHECK (channel IN ('邮件', 'WhatsApp', 'Telegram')),
    content TEXT,
    sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT '已发送' 
        CHECK (status IN ('已发送', '送达', '打开', '回复', '失败')),
    delivery_at TIMESTAMP,
    open_at TIMESTAMP,
    reply_at TIMESTAMP,
    reply_content TEXT,
    failure_reason TEXT,
    retry_count INTEGER DEFAULT 0
);
```

---

### 2.6 渠道配置表 channels

```sql
CREATE TABLE channels (
    channel_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(user_id),
    channel_name VARCHAR(100) NOT NULL,
    channel_url VARCHAR(500),
    channel_type VARCHAR(20) CHECK (channel_type IN ('B2B', '黄页', '展会', '搜索引擎')),
    priority INTEGER DEFAULT 5,
    is_enabled BOOLEAN DEFAULT TRUE,
    extra_config JSONB,  -- 额外配置
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### 2.7 关键词配置表 keywords

```sql
CREATE TABLE keywords (
    keyword_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(user_id),
    keyword VARCHAR(255) NOT NULL,
    keyword_type VARCHAR(20) CHECK (keyword_type IN ('产品词', '行业词', '组合词')),
    example_usage TEXT,
    is_enabled BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### 2.8 发送队列表 send_queue

```sql
CREATE TABLE send_queue (
    queue_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    letter_id UUID REFERENCES letters(letter_id) ON DELETE CASCADE,
    contact_id UUID REFERENCES contacts(contact_id) ON DELETE CASCADE,
    channel VARCHAR(20),
    priority INTEGER DEFAULT 0,  -- A类=3, B类=2, C类=1
    scheduled_at TIMESTAMP,  -- 计划发送时间
    status VARCHAR(20) DEFAULT '等待中' 
        CHECK (status IN ('等待中', '发送中', '已完成', '失败')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_send_queue_status ON send_queue(status);
CREATE INDEX idx_send_queue_priority ON send_queue(priority DESC);
```

## 两阶段数据关系对比

### 阶段一（飞书表格）

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│  客户主表   │──1:N──│  联系人表   │       │  开发信表   │

       └────────────────────┼────────────────────┘
                            │
                     关联: 客户ID
```

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│    用户表   │──1:N──│  客户主表   │──1:N──│  联系人表   │
│   users    │       │ customers  │       │ contacts   │
└─────────────┘       └─────────────┘       └─────────────┘
                            │
                     ┌──────┴──────┐
                     │             │
                     ▼             ▼
              ┌─────────────┐ ┌─────────────┐
              │  开发信表   │ │ 发送记录表  │
              │  letters   │ │ send_logs  │
              └─────────────┘ └─────────────┘
                     │
                     ▼
              ┌─────────────┐
              │  发送队列   │
              │ send_queue │
              └─────────────┘
```

## 数据更新规则对比

### 阶段一
```
新客户 → 老六手动录入 → 飞书表格
背调 → 老六手动补全
深挖 → 老六手动执行
开发信 → LLM生成 + 老六确认
发送 → 客户手动执行
```

### 阶段二
```
定时任务触发 → 自动搜索 → 入库
定时任务触发 → 自动背调 → 更新
定时任务触发 → 自动深挖 → 更新
自动生成 → 客户确认 → 入发送队列
定时任务 → 队列发送 → 记录状态
```
