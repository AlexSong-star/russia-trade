# 老六博客 - 技术方案

## 1. 技术选型

| 技术 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js 14+ | 已有项目经验、SEO友好、静态生成 |
| 部署平台 | Vercel | 免费额度够用、CI/CD 自动 |
| 样式方案 | Tailwind CSS | 开发快、炫酷效果容易实现 |
| 内容格式 | Markdown | 方便 AI 创作、版本控制 |
| 数据存储 | 本地文件 + JSON | 简单、免维护 |
| 评论/阅读量 | 飞书 Bitable | 强哥有飞书账号、方便管理 |

## 2. 系统架构

```
┌─────────────────────────────────────────┐
│                用户访问                  │

│             Next.js 服务                  │
│  ┌──────────┐  ┌──────────┐            │
│  │  页面    │  │  API     │            │

```

## 4. 核心功能实现

### 4.1 文章存储
- 路径：`posts/*.md`
- Frontmatter 格式：
```yaml
---
title: 文章标题
date: 2026-03-17
category: 技术
tags: [AI, 编程]
excerpt: 摘要
published: true
---
```

### 4.2 数据流程
```
Markdown 文件 ──解析──> 静态页面 ──部署──> Vercel
                         │
                         ▼
                   Bitable API
                   (评论/阅读量)
```

### 4.3 后台管理
- 路径：`/admin`
- 简单认证：环境变量设置密码
- 功能：文章管理、评论审核、数据查看

## 5. 页面路由

| 路径 | 说明 |
|------|------|
| `/` | 首页，文章列表 |
| `/posts/[slug]` | 文章详情 |
| `/category/[name]` | 分类页 |
| `/tag/[name]` | 标签页 |
| `/about` | 关于老六 |
| `/admin` | 后台管理 |

## 6. 部署方案

- **触发方式**：GitHub Push 自动部署
- **预览部署**：PR 自动生成预览链接
- **生产部署**：main 分支合并后自动部署
- **域名**：laow-blog.vercel.app（已配置）
